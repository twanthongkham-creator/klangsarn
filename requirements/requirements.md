# Requirements - ระบบจัดการคลังสารเคมี (ChemStock)

เอกสารนี้ระบุความต้องการและขอบเขตของระบบ **ChemStock (Chemical Stock Control System v2.0 Mobile-First)**

---

## 1. วัตถุประสงค์ (System Objective)
ระบบนี้ได้รับการออกแบบให้เป็นระบบบริการจัดการคลังสารเคมีแบบ Serverless ที่เน้นการใช้งานบนมือถือเป็นหลัก (Mobile-First 100%) เพื่ออำนวยความสะดวกให้เจ้าหน้าที่หน้างานสามารถบันทึกการรับเข้า-เบิกจ่าย ถ่ายภาพสารเคมี และตรวจสอบวันหมดอายุได้ทันที ณ จุดปฏิบัติงาน

---

## 2. บทบาทผู้ใช้งาน (User Roles & Permissions)

ระบบแบ่งสิทธิ์การทำงานออกเป็น 2 บทบาทหลัก:

### 2.1 ผู้ใช้ทั่วไป (General User)
- **การเข้าถึง**: ไม่ต้องล็อกอิน
- **ฟังก์ชันที่ใช้งานได้**:
  - ดูรายการสารเคมีทั้งหมดในคลัง (ผ่านตารางบน Desktop หรือการ์ดบน Mobile)
  - กรองสารเคมีตามสถานที่จัดเก็บ (ห้อง 1 ออกซิไดซ์, ห้อง 2 กรด, ห้อง 3 ด่าง)
  - บันทึกธุรกรรม **รับเข้า (IN)** และ **เบิกจ่าย (OUT)** ของสารเคมีที่มีอยู่ในคลัง
  - ดูประวัติการทำรายการรับ-จ่ายทั้งหมดพร้อมระบบค้นหาและกรองข้อมูล
  - ดูแดชบอร์ดสรุปสถิติ สัดส่วนสารเคมีตามสถานที่จัดเก็บ และการแจ้งเตือนใกล้หมดอายุ

### 2.2 ผู้ดูแลระบบ (Administrator)
- **การเข้าถึง**: เข้าผ่านหน้า Admin Panel ด้วยรหัสผ่าน (รหัสผ่านเริ่มต้น: `chem@admin`)
- **ฟังก์ชันที่ใช้งานได้ (เพิ่มเติมจากสิทธิ์ผู้ใช้ทั่วไป)**:
  - **เพิ่มรายการใหม่ (Create)**: เพิ่มสารเคมีใหม่เข้าสู่คลัง (ระบุชื่อ, CAS Number, สถานที่เก็บ, จำนวนคงเหลือ, หน่วยนับ, วันผลิต/วันหมดอายุ และรูปภาพสูงสุด 3 รูป)
  - **แก้ไขรายการ (Update)**: แก้ไขข้อมูลสารเคมีทุกประเภท รวมถึงการลบ/เพิ่มรูปภาพประกอบ
  - **ลบรายการ (Delete)**: ลบสารเคมีออกจากสต็อก (จะลบประวัติการรับ-จ่ายที่เกี่ยวข้องโดยอัตโนมัติ)
  - **การจัดการประวัติ**: สามารถล้างประวัติการทำรายการทั้งหมดระบบ (Clear Transactions)
  - **ส่งออกข้อมูล (Export)**: ดาวน์โหลดข้อมูลสารเคมีทั้งหมด หรือข้อมูลประวัติการรับ-จ่าย เป็นไฟล์ CSV
  - **ตั้งค่าความปลอดภัย**: เปลี่ยนรหัสผ่านของผู้ดูแลระบบ

---

## 3. โครงสร้างฐานข้อมูล (Database Schema)

ระบบใช้งานฐานข้อมูล PostgreSQL บนบริการ Supabase ผ่าน Serverless Client JS SDK ประกอบด้วย 2 ตารางหลัก ดังนี้:

### 3.1 ตาราง `chemical_stock` (คลังสารเคมีหลัก)
| Column Name | Data Type | Constraint | Description |
| :--- | :--- | :--- | :--- |
| `id` | bigint / uuid | Primary Key, Auto Increment | รหัสสารเคมี |
| `chemical_name` | text | Not Null | ชื่อสารเคมี (เช่น Sodium Hydroxide) |
| `material_number` | text | Nullable | รหัสวัสดุ / รหัสสินค้า (Material Number) |
| `lot_number` | text | Nullable | เลขที่ล็อตการผลิต (Lot / Batch Number) |
| `quantity` | double precision | Not Null, Default `0.00` | จำนวนคงเหลือปัจจุบัน |
| `unit` | text | Not Null | หน่วยนับ (เช่น kg, L, ขวด) |
| `mfg_date` | date | Nullable | วันผลิต (Manufacturing Date) |
| `exp_date` | date | Nullable | วันหมดอายุ (Expiry Date) |
| `location` | text | Not Null | ห้องจัดเก็บสารเคมี |
| `price_per_unit` | double precision | Nullable, Default `0.0` | ราคาซื้อต่อหน่วย |
| `image_urls` | text (JSON Array) | Nullable | เก็บรูปภาพประกอบ (ในรูปแบบ Array ของ Base64 Data URL) |
| `created_at` | timestamp | Default `now()` | วันเวลาที่สร้างรายการ |

### 3.2 ตาราง `chemical_transactions` (ประวัติรับเข้า-เบิกจ่าย)
| Column Name | Data Type | Constraint | Description |
| :--- | :--- | :--- | :--- |
| `id` | bigint | Primary Key, Auto Increment | รหัสธุรกรรม |
| `chemical_id` | bigint | Foreign Key -> `chemical_stock.id` (On Delete Cascade) | ลิงก์ไปยังสารเคมีที่ทำรายการ |
| `type` | text | Not Null (ค่าที่เป็นไปได้: `IN`, `OUT`) | ประเภทธุรกรรม (รับเข้า หรือ เบิกจ่าย) |
| `quantity` | double precision | Not Null | จำนวนที่มีการรับเข้าหรือเบิกจ่าย |
| `remark` | text | Nullable | หมายเหตุ / แผนกที่เบิก / ผู้รับผิดชอบ |
| `price_per_unit` | double precision | Nullable, Default `0.0` | ราคาต่อหน่วย ณ วันที่ทำรายการ |
| `free_quantity` | double precision | Nullable, Default `0.0` | จำนวนสารเคมีที่ได้รับแถมฟรี (โปรโมชั่น) |
| `saving` | double precision | Nullable, Default `0.0` | มูลค่าส่วนลด/ประหยัดที่เกิดขึ้น |
| `transaction_date` | timestamp | Default `now()` | วันเวลาที่เกิดธุรกรรม |

## 4. ความต้องการเชิงฟังก์ชันที่สำคัญ (Key Functional Requirements)

### 4.1 การบีบอัดและจัดเก็บรูปภาพ (Image Compression)
- ระบบต้องรองรับการอัปโหลดภาพสูงสุด 3 รูปต่อหนึ่งรายการสารเคมี
- เพื่อประหยัดพื้นที่ฐานข้อมูลและช่วยให้โหลดหน้าเว็บบนมือถือได้เร็ว รูปภาพที่อัปโหลดต้องถูกแปลงและบีบอัดผ่าน HTML5 Canvas:
  - ขนาดความกว้างสูงสุดไม่เกิน `600px`
  - แปลงประเภทเป็น `image/jpeg`
  - คุณภาพการบีบอัด `72%`
  - บันทึกในรูปแบบ Base64 ลงในฟิลด์ `image_urls`

### 4.2 ระบบแจ้งเตือนวันหมดอายุ (Expiry Alerts)
- ระบบต้องคำนวณและแสดงสีสถานะของวันหมดอายุแบบเรียลไทม์:
  - **หมดอายุแล้ว (Expired)**: แสดงสถานะสีแดง แจ้งว่า "หมดอายุ"
  - **ใกล้หมดอายุ (Near Expiry)**: เหลือวันหมดอายุน้อยกว่าหรือเท่ากับ 30 วัน แสดงสถานะสีส้ม แจ้งเตือนจำนวนวันที่เหลือ
  - **ปกติ**: วันหมดอายุมากกว่า 30 วัน แสดงสถานะสีเขียว (หรือปกติ)
- บนหน้าหลักและหน้าแดชบอร์ดต้องแสดงสถิติจำนวนสารเคมีที่เข้าข่ายใกล้หมดอายุ/หมดอายุ เพื่อให้ตรวจสอบได้รวดเร็ว

### 4.3 ระบบบันทึกจำนวนสต็อกอัตโนมัติ (Stock Auto-update)
- เมื่อมีการบันทึกธุรกรรมรับเข้า (IN) จำนวนคงเหลือใน `chemical_stock` จะต้องเพิ่มขึ้นโดยอัตโนมัติ
- เมื่อมีการเบิกจ่าย (OUT) จำนวนคงเหลือใน `chemical_stock` จะต้องลดลงโดยอัตโนมัติ
- **ระบบป้องกันข้อผิดพลาด**: หากจำนวนที่เบิกจ่าย (OUT) มากกว่าจำนวนคงเหลือในสต็อก ระบบต้องแสดงข้อความแจ้งเตือนด้วยสีแดงว่า "สต็อกคงเหลือไม่เพียงพอ!" และไม่อนุญาตให้บันทึกธุรกรรมนั้น

## 5. ความต้องการเชิงเทคนิคและสถาปัตยกรรม (Technical Requirements)
- **Frontend**: พัฒนาด้วยเทคนิค Single Page Logic per View โดยใช้ Vanilla JavaScript และ HTML5 ไร้การพึ่งพา Framework ขนาดใหญ่เพื่อความรวดเร็วในการโหลด
- **UI & Styling**: ใช้ Bootstrap 5.3.3 ร่วมกับ CSS Custom Variable เพื่อรองรับการแสดงผลหน้าจอมือถือ (Responsive)
- **Data Visualization**: ใช้ `Chart.js` วาดกราฟโดนัท (Doughnut Chart) แสดงสัดส่วนการจัดเก็บสารเคมีแยกตามสถานที่
- **Backend & Database**: ใช้ Supabase เป็นบริการฐานข้อมูลแบบ Serverless ผ่าน API Direct Connection (ไม่ต้องมี Node.js backend)
- **SEO & Performance**: หน้าเว็บแยกส่วนกันอย่างเหมาะสม มีความปลอดภัยด้าน Session สำหรับสิทธิ์ผู้ดูแลระบบ (จัดเก็บด้วย LocalStorage)

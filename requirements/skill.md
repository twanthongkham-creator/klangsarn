## 1. การตั้งค่าระบบสำหรับพัฒนาต่อยอด (Local Environment Setup)

เนื่องจากระบบนี้เป็น Serverless Static Web Application (Vanilla JS + Supabase) คุณไม่จำเป็นต้องติดตั้ง Node.js หรือ Compiler ใดๆ แต่สามารถรันบน Web Server ทั่วไปได้ทันที:

### 1.1 การทดสอบผ่าน XAMPP (Apache Web Server)
1. ติดตั้ง **XAMPP** บนเครื่องคอมพิวเตอร์ของคุณ
2. นำโฟลเดอร์โปรเจกต์ `matchem` ไปวางที่ไดเรกทอรี `c:\xampp\htdocs\matchem`
3. เปิดโปรแกรม XAMPP Control Panel และกด Start บริการ **Apache**
4. เปิดเว็บเบราว์เซอร์ไปที่ลิงก์: [http://localhost/matchem/index.html](http://localhost/matchem/index.html)

## 2. การจัดเตรียมและสร้างตารางบน Supabase (Database Initialization Script)

หากต้องการนำระบบนี้ไปใช้ใน Project ใหม่ ให้สร้างตาราง (Tables) บน Supabase SQL Editor โดยใช้สคริปต์ SQL ดังต่อไปนี้:

```sql
-- 1. สร้างตารางเก็บรายการสารเคมี (chemical_stock)
CREATE TABLE chemical_stock (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    chemical_name text NOT NULL,
    material_number text,
    lot_number text,
    quantity double precision NOT NULL DEFAULT 0.0,
    unit text NOT NULL,
    mfg_date date,
    exp_date date,
    location text NOT NULL,
    price_per_unit double precision DEFAULT 0.0,
    image_urls text, -- จัดเก็บ Base64 ในรูปแบบ JSON String [ "base64_img1", "base64_img2" ]
    created_at timestamp with time zone DEFAULT now() NOT NULL
);

-- 2. สร้างตารางเก็บประวัติการทำธุรกรรม (chemical_transactions)
CREATE TABLE chemical_transactions (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    chemical_id bigint REFERENCES chemical_stock(id) ON DELETE CASCADE,
    type text NOT NULL, -- 'IN' หรือ 'OUT'
    quantity double precision NOT NULL,
    remark text,
    price_per_unit double precision DEFAULT 0.0,
    free_quantity double precision DEFAULT 0.0,
    saving double precision DEFAULT 0.0,
    transaction_date timestamp with time zone DEFAULT now() NOT NULL
);
```

## 3. ความรู้และเทคนิคหลักใน Codebase (Core Development Skills)

นักพัฒนาจำเป็นต้องทำความเข้าใจกลไกและรูปแบบโค้ดที่สำคัญ 3 เรื่องในโปรเจกต์นี้ก่อนแก้ไขโค้ด:

### 3.1 การสืบค้นข้อมูลข้ามตารางผ่าน Supabase Client (Relational Queries)
ในระบบมีการสร้าง Relation เชื่อมโยงประวัติรับ-จ่ายกับข้อมูลสารเคมีหลัก การเรียกค้นข้อมูลให้ได้ข้อมูลของสารเคมีและหน่วยมาด้วย ให้ใช้รูปแบบการเลือกดังนี้:

```javascript
// ตัวอย่างการคิวรีข้อมูล Relational Join
const { data, error } = await _supabase
    .from('chemical_transactions')
    .select('id, type, quantity, remark, transaction_date, chemical_stock(chemical_name, unit)')
    .order('transaction_date', { ascending: false });
```

### 3.2 ระบบบีบอัดและจัดการไฟล์ภาพ (HTML5 Canvas Compression)
เพื่อจำกัดการใช้งานพื้นที่เก็บข้อมูล รูปภาพจะถูกย่อขนาดที่ฝั่งผู้ใช้งาน (Client-Side) ก่อนบันทึกลงฐานข้อมูล:

```javascript
// ฟังก์ชันย่อภาพและแปลงเป็น Base64 (อยู่ใน js/script.js)
const reader = new FileReader();
reader.readAsDataURL(file);
reader.onload = ev => {
    const img = new Image();
    img.src = ev.target.result;
    img.onload = () => {
        const canvas = document.createElement('canvas');
        const MAX_W = 600; // จำกัดความกว้างสูงสุด 600px
        let w = img.width, h = img.height;
        if (w > MAX_W) { h *= MAX_W / w; w = MAX_W; }
        canvas.width = w; canvas.height = h;
        canvas.getContext('2d').drawImage(img, 0, 0, w, h);
        
        // บีบอัดภาพเป็น jpeg ด้วยคุณภาพ 72%
        const b64 = canvas.toDataURL('image/jpeg', 0.72);
        uploadedImagesBase64.push(b64);
    };
};
```

### 3.3 ระบบการล็อกอินของ Admin (Admin Session Management)
เพื่อรักษาความปลอดภัยขั้นพื้นฐาน ระบบควบคุมสิทธิ์ผ่าน LocalStorage:
- **ตัวแปรสิทธิ์ Admin**: `chemstock_admin` (เก็บค่าเป็น `'true'` เมื่อเข้าสู่ระบบสำเร็จ)
- **ตัวแปรรหัสผ่าน**: `chemstock_admin_pwd` (รหัสผ่านเริ่มต้นหากไม่มีการเปลี่ยนคือ `chem@admin`)
- **การอัปเดต UI**: การซ่อนหรือแสดงปุ่มควบคุมแบบ Admin จะขึ้นกับค่าที่รีเทิร์นจากฟังก์ชัน `isAdminLoggedIn()` ในไฟล์ `js/ui.js`
